Basics

  • filter: pick rows
  • arrange: reorder rows
  • select: pick cols/vars
  • mutate: add new columns (derived vars)
  • summarize: reduce many values to scalar

Workflow

  • Start wtih data
  • Each layer with basic functions produces a new data frame
  • Multiple layers added together as pipeline
  • Similar to ggplot2

                    (dec25 <- filter(flights, month == 12, day == 25))
                    #> # A tibble: 719 x 19
                    #>    year month   day dep_time sched_dep_time dep_delay arr_time sched_arr_time
                    #>   <int> <int> <int>    <int>          <int>     <dbl>    <int>          <int>
                    #> 1  2013    12    25      456            500        -4      649            651
                    ...
                    #> # … with 713 more rows, and 11 more variables: arr_delay <dbl>, carrier <chr>,
                    #> #   flight <int>, tailnum <chr>, origin <chr>, dest <chr>, air_time <dbl>,
                    #> #   distance <dbl>, hour <dbl>, minute <dbl>, time_hour <dttm>

Logical Operators

  • &: and
  • |: or
  • !: not
  • Not &&, ||, wich produces scalars

Missing Values

  • NA: missing values
  • filter: excludes both FALSE and NA values 

                    NA == NA
                    #> [1] NA
                    filter(df, is.na(x) | x > 1)
                    #> # A tibble: 2 x 1
                    #>       x
                    #>   <dbl>
                    #> 1    NA
                    #> 2     3

arrange

  • Default is ascending order
  • Missing values at the end
  • Ordered by variable ordered passed in

                    arrange(flights, year, month, day)
                    #> # A tibble: 336,776 x 19
                    ...
                    arrange(flights, desc(dep_delay))
                    #> # A tibble: 336,776 x 19

select

  • Pick columns/variables
  • Powerful when name patterns are passed
  • Built-in name pattern functions

                    # Select columns by name
                    select(flights, year, month, day)
                    # Select all columns between year and day (inclusive)
                    select(flights, year:day)
                    # Select all columns except those from year to day (inclusive)
                    select(flights, -(year:day))

Helper Functions

  • starts_with('abc'): match var names begin with 'abc'
  • ends_with('abc'): end with 'abc'
  • contains('abc'): contain 'abc'
  • matches('(.)\\1'): match regular expression
  • num_range("x", 1:3): x1, x2, and x3

rename

rename changes variable names


                    rename(flights, tail_num = tailnum)

everything

everything() refers to all (other) variable names


                    select(flights, time_hour, air_time, everything())

Moves two variables to the first two columns

mutate

  • Transformed variables added to the end of tibble
  • Vector-type operations
  • Refer variable names without quotes
  • You can refer to columns that just created

                flights_sml <- select(flights, 
                    year:day, 
                    ends_with("delay"), 
                    distance, 
                    air_time
                )
                mutate(flights_sml,
                    gain = dep_delay - arr_delay,
                    speed = distance / air_time * 60
                )
                mutate(flights_sml,
                    gain = dep_delay - arr_delay,
                    hours = air_time / 60,
                    gain_per_hour = gain / hours
                )

transmute

Unlike mutate, transmute only keeps the transformed variables


                transmute(flights,
                    gain = dep_delay - arr_delay,
                    hours = air_time / 60,
                    gain_per_hour = gain / hours
                  )
                  #> # A tibble: 336,776 x 3
                  #>    gain hours gain_per_hour

Operators in mutate/transmute

  • Most operators should work
  • Operations are vectorized
  • Some specical ones

                    y <- c(1, 2, 2, NA, 3, 4)
                    min_rank(y)

summarise

  • mutate produces same number of rows
  • summarise reduces to a single row
  • Usually combined with group_by, to produce summaries for each group
    • ungroup removes grouping
  • Only keeps the summarized variables, like transmute
  • Most summary functions work and more (check text book)

                    by_day <- group_by(flights, year, month, day)
                    summarise(by_day, delay = mean(dep_delay, na.rm = TRUE))
                    #> `summarise()` regrouping output by 'year', 'month' (override with `.groups` argument)
                    #> # A tibble: 365 x 4
                    #> # Groups:   year, month [12]
                    #>    year month   day delay

Pipe %>%


                    delays <- flights %>% 
                    group_by(dest) %>% 
                    summarise(
                      count = n(),
                      dist = mean(distance, na.rm = TRUE),
                      delay = mean(arr_delay, na.rm = TRUE),
                      n = n()
                    ) %>% 
                    filter(count > 20, dest != "HNL")
                    #> `summarise()` ungrouping output (override with `.groups` argument)
                    ggplot(data = delays, mapping = aes(x = dist, y = delay)) +
                    geom_point(aes(size = count), alpha = 1/3) +
                    geom_smooth(se = FALSE)

tibble

  • Much like data.frame but does less
  • Never converts strings to factors
  • Never changes var names
  • never creates row names
  • as_tibble, tibble
  • Pretty much like data.frame and other packages, if not, convert back to data.frame

tribble


                    tribble(
                        ~x, ~y, ~z,
                        #--|--|----
                        "a", 2, 3.6,
                        "b", 1, 8.5
                      )
                      #> # A tibble: 2 x 3
                      #>   x         y     z
                      #>   <chr> <dbl> <dbl>
                      #> 1 a         2   3.6
                      #> 2 b         1   8.5

Tidy Data

  • Tidy data have obs as rows, var as cols
  • Common problems:
    • One var spread across multiple cols
    • One obs scattered across multiple rows
  • Typical in repeated measures/longitudinal studies

Longer


                    table4a
                    #> # A tibble: 3 x 3
                    #>   country     `1999` `2000`
                    #> * <chr>        <int>  <int>
                    #> 1 Afghanistan    745   2666

                    table4a %>% 
                    pivot_longer(c(`1999`, `2000`), names_to = "year", values_to = "cases")
                    #> # A tibble: 6 x 3
                    #>   country     year   cases
                    #>   <chr>       <chr>  <int>
                    #> 1 Afghanistan 1999     745

` backtick used for non-syntactic var names starting with numbers, interpreted as strings, names_to is the new var name for cols, values_to is the var name for cell values, quotes because they don't exist yet

Wider


                    table2
                    #> # A tibble: 12 x 4
                    #>   country      year type           count
                    #>   <chr>       <int> <chr>          <int>
                    #> 1 Afghanistan  1999 cases            745
                    
                    table2 %>%
                    pivot_wider(names_from = type, values_from = count)
                    #> # A tibble: 6 x 4
                    #>   country      year  cases population
                    #>   <chr>       <int>  <int>      <int>
                    #> 1 Afghanistan  1999    745   19987071

pivot_longer and pivot_wider complements each other

separate


                    table3
                    #> # A tibble: 6 x 3
                    #>   country      year rate             
                    #> * <chr>       <int> <chr>            
                    #> 1 Afghanistan  1999 745/19987071   
                       
                    table3 %>% 
                    separate(rate, into = c("cases", "population"))
                    #> # A tibble: 6 x 4
                    #>   country      year cases  population
                    #>   <chr>       <int> <chr>  <chr>     
                    #> 1 Afghanistan  1999 745    19987071

                    table3 %>% 
                    separate(rate, into = c("cases", "population"), sep = "/")

Split by non-alphanumeric characters, e.g. / here. by default, variables are splited like strings and saved as characters

dplyr vs base

dplyr merge
inner_join(x, y) merge(x, y)
left_join(x, y) merge(x, y, all.x = TRUE)
right_join(x, y) merge(x, y, all.y = TRUE),
full_join(x, y) merge(x, y, all.x = TRUE, all.y = TRUE)

Strings

String Functions

  • str_c: combining strings
  • str_sub: subsetting strings
  • str_to_upper/str_to_lower: convert to upper/lower cases
  • str_sort: sort
  • str_view/str_view_all: regular expressions for one or all matches
  • str_extract: extract matches by REs
  • str_split: split into pieces

Example


                    colours <- c("red", "orange", "yellow", "green", "blue", "purple")
                    colour_match <- str_c(colours, collapse = "|")
                    colour_match
                    #> [1] "red|orange|yellow|green|blue|purple"                   
                    str_extract(more, colour_match)
                    #> [1] "blue"   "green"  "orange"

Factors

  • Works with basic factors in R
  • Easier to reorder levels using fct_reorder
  • Change reference level by fct_relevel
  • Modify levels by fct_recode for printing or graphing
  • Lump small levels together by fct_lump 

                    relig_summary <- gss_cat %>%
                    group_by(relig) %>%
                    summarise(
                      age = mean(age, na.rm = TRUE),
                      tvhours = mean(tvhours, na.rm = TRUE),
                      n = n()
                    )
                  #> `summarise()` ungrouping output (override with `.groups` argument)                  
                  ggplot(relig_summary, aes(tvhours, relig)) + geom_point()
                

                    ggplot(relig_summary, aes(tvhours, fct_reorder(relig, tvhours))) +
                    geom_point()
                    ## OR
                    relig_summary %>%
                    mutate(relig = fct_reorder(relig, tvhours)) %>%
                    ggplot(aes(tvhours, relig)) +
                        geom_point()
