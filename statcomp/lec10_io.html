IO

  • R provides many IO capabilities
  • Versatile function scan for text and internet, but it reads all at once
  • readline read in a single line from keyboard or file
  • cat slightly more customizable than print

Text vs Binary File

  • Text file: ASCII characters or other coding, e.g. UTF-8
  • Binary: 0/1 with specific formats (e.g. jpeg)
  • We will focus on text files
  • Reading binary files can be built on other C libraries

Connection

  • Reading files starts with opening connections file
  • While open, readLines reading one line at a time and move to next line
  • seek rewind to any position of the file
  • Remember to close connections
  • Writing files is similar, replacing readLines with writeLines

Example


                    > conn <- file("z1", "r")
                    > readLines(conn, n=2)
                    [1] "line 1" "line 2"
                    > seek(con=conn, where=0)
                    > readLines(conn, n=1)
                    [1] "line 1"

TCP/IP

  • IP address: unique identifier for your device (intra vs internet)
  • DNS: translate addresses to IPs, e.g. google.com
  • Port: different applications uses specific ports
  • Socket: connection between device A and B
  • Client/Server model: servers listening incoming requests from clients on certin port and sending responses back

Functions

  • readLines, writeLines: receive and send line by line
  • serialize, unserialize: convert R object to chracter string and send
  • readBin, writeBin: sending in binary form
  • Other functions: socketConnection, socketSelect
